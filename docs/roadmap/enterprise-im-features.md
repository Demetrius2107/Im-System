## 一、消息系统

### 1.1 消息类型

| 消息类型 | 说明 | 大厂实现要点 |
|---------|------|-------------|
| **文本消息** | 普通文字、Markdown、富文本 | 支持 @提及、链接预览、表情解析 |
| **图片消息** | JPEG/PNG/WebP/GIF | 缩略图+原图分离、CDN加速、WebP压缩、 Progressive JPEG |
| **语音消息** | AMR/AAC/Opus | 语音转文字(ASR)、波形可视化、倍速播放 |
| **视频消息** | MP4/H.264/H.265 | 首帧封面、分片上传、转码适配、HLS播放 |
| **文件消息** | 任意格式 | 分片上传、秒传(MD5)、断点续传、在线预览 |
| **位置消息** | 经纬度 | 地图缩略图、POI名称、导航跳转 |
| **红包/转账** | 虚拟资产 | 一致性事务、防并发、账单记录 |
| **自定义消息** | 业务扩展 | 通过 JSON/Protobuf 扩展，客户端自定义渲染 |

### 1.2 消息可靠性

| 机制 | 说明 | 实现方式 |
|------|------|---------|
| **消息去重** | 防止重复发送 | 客户端生成 messageId (UUID/Snowflake)，服务端幂等校验 |
| **消息ACK** | 确认送达 | 接收方收到后回复 ACK，超时未 ACK 则重推 |
| **消息序号** | 全局有序 | 每个会话维护一个 sequence，单调递增，用于排序和去重 |
| **消息持久化** | 不丢消息 | 先写DB再推送，写DB失败则不推送 |
| **离线消息** | 上线补拉 | Redis ZSet 存储，按 sequence 拉取，拉取后清除 |
| **消息漫游** | 多端同步 | 服务端存储全量消息历史，新设备登录后拉取 |
| **已读回执** | 显示已读/未读 | 单聊实时回执，群聊聚合已读人数 |
| **消息撤回** | 限定时间内撤回 | 2分钟内可撤回，撤回后显示"对方已撤回一条消息" |

### 1.3 消息存储策略

```java
// 大厂常见分表策略
im_message_body_0      // 消息体（按 message_key 哈希）
im_message_history_0   // 消息索引（按 owner_id 哈希分表）
im_group_message_0     // 群消息（按 group_id 哈希分表）

// 分表数量：64/128/256 张，按业务量动态调整
// 分库策略：按 app_id 分库
```

## 六、消息推送

### 6.1 推送通道

| 通道 | 适用场景 | 实现方式 |
|------|---------|---------|
| **TCP长连接** | 在线消息 | Netty、WebSocket、自研协议 |
| **APNs/FCM** | iOS离线 | Apple Push / Firebase Cloud Messaging |
| **华为/小米/OPPO推送** | Android离线 | 厂商通道、保活机制 |
| **WebSocket** | Web端 | 标准WS协议，浏览器兼容 |
| **SSE** | 服务端推送 | Server-Sent Events，单向推送 |

### 6.2 推送策略

| 策略 | 说明 |
|------|------|
| 在线优先 | 在线设备通过长连接推送，不触发离线推送 |
| 离线转推送 | 离线后转为 APNs/厂商推送，仅推摘要 |
| 聚合推送 | 多条消息合并为一条通知，减少打扰 |
| 静默推送 | 仅更新角标，不弹通知 |
| 免打扰时段 | 夜间/会议/自定义时段不推送 |

## 八、存储与缓存

### 8.1 缓存分层

```
L1: 本地内存缓存 (Caffeine/Guava)
    ├── 用户session: 5分钟过期
    ├── 好友关系链: 10分钟过期
    └── 群组成员: 5分钟过期

L2: 分布式缓存 (Redis Cluster)
    ├── 用户在线状态: Hash <appId:userSession:userId>
    ├── 离线消息: ZSet <appId:offlineMessage:userId>
    ├── 消息去重: String <appId:cacheMessage:messageId>
    ├── 序列生成器: String <seq:xxx>
    └── 会话列表: Hash <appId:conversation:userId>

L3: 持久化存储 (MySQL/MySQL Cluster)
    ├── 用户数据: im_user_data
    ├── 好友关系: im_friendship
    ├── 群组数据: im_group, im_group_member
    ├── 消息体: im_message_body (分表)
    ├── 消息历史: im_message_history (分表)
    └── 会话设置: im_conversation_set
```

### 8.2 数据库分片策略

```sql
-- 消息体分表: message_body_0 ~ message_body_63
-- 分片键: message_key % 64

-- 消息历史分表: message_history_0 ~ message_history_63
-- 分片键: (app_id + owner_id) % 64

-- 群消息分表: group_message_0 ~ group_message_63
-- 分片键: group_id % 64
```

## 九、高可用架构

### 9.1 网关层

| 组件 | 说明 |
|------|------|
| 负载均衡 | Nginx/ALB → 网关集群 |
| 连接管理 | 百万长连接管理，连接迁移 |
| 协议适配 | TCP/WS/HTTP 统一接入 |
| 限流熔断 | 令牌桶 + 滑动窗口 + Sentinel/Hystrix |
| 防攻击 | DDoS防护、连接频率限制、IP黑名单 |

### 9.2 服务层

| 组件 | 说明 |
|------|------|
| 服务注册 | ZooKeeper/Nacos/Consul |
| 服务发现 | 客户端发现/服务端发现 |
| 负载均衡 | 一致性哈希(保持session)、轮询、加权 |
| 熔断降级 | Sentinel/Resilience4j |
| 链路追踪 | SkyWalking/Pinpoint/Zipkin |
| 配置中心 | Apollo/Nacos/Spring Cloud Config |

### 9.3 消息队列

```
生产端 → RocketMQ/Kafka → 消费端
├── 消息可靠性: 同步刷盘 + 主从复制
├── 消息顺序: 同一会话的消息发往同一分区
├── 消息重试: 失败后重试3次，进入死信队列
├── 消息去重: 消费端幂等，按messageId去重
└── 消息回溯: 支持按时间戳回溯消费
```

## 十二、推荐技术栈对比
