## 架构演进路线

### 当前架构 (v1.0)
```
单体业务服务 (im-service) + 独立网关 (im-tcp) + 独立存储 (im-message-store)
```

### 目标架构 (v2.0)
```
                   ┌──────────┐
                   │ API网关   │
                   │ (Kong/Nginx)
                   └────┬─────┘
                        │
          ┌─────────────┼─────────────┐
          │             │             │
     ┌────▼───┐   ┌────▼───┐   ┌────▼───┐
     │用户服务  │   │好友服务  │   │群组服务  │  ← 按领域拆分微服务
     └────┬───┘   └────┬───┘   └────┬───┘
          │             │             │
     ┌────▼─────────────▼─────────────▼───┐
     │           消息服务 (Message Service) │
     │       P2P / 群聊 / 离线 / 搜索      │
     └────────────────────────────────────┘
```

### 演进步骤

1. **Step 1**: 完善单体内聚，抽取 Repository 接口
2. **Step 2**: 按领域拆分 Maven Module，独立部署
3. **Step 3**: 引入 API 网关，统一鉴权/限流
4. **Step 4**: 服务网格化，引入 Service Mesh

## 技术债务清单

| 类别 | 项目 | 优先级 | 说明 |
|------|------|--------|------|
| 测试 | 单元测试缺失 | 🔴 P0 | 无任何 Java 单元测试 |

| 规范 | 注释覆盖率不足 | 🟡 P1 | 部分类缺少类级注释 |
| 性能 | 未做连接池调优 | 🟢 P2 | Netty/Redis/DB 连接池参数 |
| 监控 | 无 APM 接入 | 🟢 P2 | 无 SkyWalking/Pinpoint |
