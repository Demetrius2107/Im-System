## 四、数据流架构

### 消息发送流程 (P2P)

```
Client A                    im-tcp                   im-service              im-message-store
   │                          │                         │                        │
   │── TCP/WS 消息 ──────────▶│                         │                        │
   │                          │── Feign 鉴权 ──────────▶│                        │
   │                          │◀───── ResponseVO ───────│                        │
   │                          │                         │                        │
   │                          │── MQ: 消息内容 ────────▶│                        │
   │                          │                         │── MQ: 存储请求 ────────▶│
   │                          │                         │                        │── MySQL写入
   │                          │                         │◀───── ACK ─────────────│
   │                          │◀──── MQ: 消息同步 ──────│                        │
   │◀───── ACK ───────────────│                         │                        │
   │                          │                         │                        │
   │                          │── MQ: 消息分发 ────────▶│                        │
   │                          │                         │── MQ: 推送 Client B ──▶│
   │◀──── 同步给其他端 ───────│                         │                        │
```

### 离线消息存储

```
1. 消息发送 → 在线接收方不在线
2. 存入 Redis ZSet (Sorted Set)
   Key: appId:offlineMessage:userId
   Score: messageSequence
   Value: OfflineMessageContent JSON
3. 用户上线 → 拉取离线消息
   ZRANGEBYSCORE key lastSequence maxSeq
4. 同步完成后清除已拉取的离线消息
```
